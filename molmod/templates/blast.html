What does this mean? This corresponds to a nucleotide BLAST (BLASTn) search filtered on -perc_identity and -qcov_hsp_perc. Query cover is calculated per High Scoring Pair (HSP), i.e. per ungapped local pairwise alignment between query and subject, and we only report the best HSP for each query-subject hit. Result fields correspond to qacc, stitle, pident, qcovhsp and evalue in tabular BLASTn output format (â€“outfmt=6), and rows are sorted on Query and E-value by default. E-value specifies the number of hits of similar (or higher) score that we expect to see by chance, i.e. smaller values indicate 'better' hits.
